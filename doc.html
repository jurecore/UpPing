Documentación UpPing

Descripción

Este proyecto es una aplicación de monitoreo de dominios que permite verificar el estado y el tiempo de respuesta de diferentes dominios o direcciones IP.

Dependencias

Dependencias

  • express: Framework web para Node.js que facilita la creación de aplicaciones web y APIs.
    npm install express
  • mongoose: ODM (Object Data Modeling) para MongoDB y Node.js, que proporciona una solución basada en esquemas para modelar los datos de la aplicación.
    npm install mongoose
  • body-parser: Middleware para manejar datos de formularios y JSON en las solicitudes HTTP.
    npm install body-parser
  • dotenv: Carga variables de entorno desde un archivo .env a process.env.
    npm install dotenv
  • ping: Librería para hacer ping a direcciones IP o dominios.
    npm install ping

Dependencias de Desarrollo

  • nodemon: Herramienta que ayuda a desarrollar aplicaciones basadas en Node.js al reiniciar automáticamente la aplicación cuando se detectan cambios en los archivos.
    npm install --save-dev nodemon

Uso

Para iniciar el proyecto, asegúrate de tener todas las dependencias instaladas y luego ejecuta:

npm start

Arquitectura del Proyecto

La arquitectura del proyecto está organizada de la siguiente manera:

  • app.js: Archivo principal de la aplicación que configura el servidor Express y las rutas.
  • modelos/: Contiene los modelos de datos de Mongoose.
    • Dominio.js: Modelo que define la estructura de los documentos de dominio en la base de datos.
  • controladores/: Contiene la lógica de negocio y las funciones que manejan las solicitudes.
    • dominioControlador.js: Controlador que maneja las operaciones relacionadas con los dominios, como agregar, obtener y eliminar dominios.
  • rutas/: Define las rutas de la API y las conecta con los controladores.
    • dominioRutas.js: Archivo que define las rutas relacionadas con los dominios.
  • public/: Contiene los archivos estáticos que se sirven al cliente, como HTML, CSS y JavaScript.
    • styles/: Carpeta que contiene los archivos CSS para el estilo de la aplicación.
    • js/: Carpeta que contiene los archivos JavaScript para la funcionalidad del lado del cliente.
    • img/: Carpeta que contiene las imágenes utilizadas en la aplicación.
    • dominios.html: Página que muestra la lista de dominios.
    • detalle.html: Página que muestra los detalles de un dominio específico.
    • index.html: Página de inicio de la aplicación.
